# Arena 4×4: Hand off contacts to a J Way newsletter

Date: 2026-06-17
Status: Approved (design)

## Goal
When a contact finishes the Arena 4×4, keep nurturing them on "The J Way"
newsletter (the existing plain-text, personal monthly note). Two paths,
split by whether the contact replied during the 4×4.

## Background
- The 4×4 is the `fourbyfour` AICB preset. It sends each contact four
  emails on a schedule.
- "The J Way" is an existing newsletter STYLE (`newsletter_style == "j_way"`,
  generated by `_generate_jway_newsletter`, ~L43760): plain-text, personal,
  live BLS highlights + available talent. The other style is "Full Send".
- Replies are already detected and recorded via `add_responded` /
  `responded.json` (~L5046-5077); reply detection also cancels a
  responder's remaining 4×4 emails.
- The Responses page (`p_responses`, ~L13797) already has a per-responder
  "enroll in Slow Drip / Newsletter" action (~L14111) and a "Graduate All
  Responded" bulk action (~L26831).
- Newsletters are campaigns flagged by `market_analysis`; contacts join at
  the next upcoming issue.

## Scope

### Path A — Non-responders (automatic, zero-config)
When a contact's FINAL 4×4 email (the last `email_auto` step) is sent and
that contact is NOT in `responded.json`, auto-enroll them into the
designated J Way handoff newsletter, joining at the next issue.

- Zero configuration: the first auto-handoff creates ONE default J Way
  newsletter (`newsletter_style = "j_way"`), seeded from the rep's sector
  and region, and every later handoff reuses it. There is deliberately NO
  "pick the auto target" setting in this version (see Out of scope).
- Guards: skip if the contact replied (in `responded.json`), is on the DNC
  list, or is already enrolled in that newsletter.
- Only fires for `fourbyfour` campaigns.

### Path B — Responders (manual, rep's choice)
A reply already lands the contact in the responded list. From there the rep
gets a per-contact "Add to newsletter" action offering two choices:
- Add to an EXISTING newsletter (any the rep already runs, J Way or Full
  Send), or
- Create a NEW J Way newsletter inline (seeded from sector/region), then
  enroll the contact into it.

This reuses the existing responder-enroll picker (~L14111) and adds the
"create new J Way" option. The action must be reachable from the responded
list, not only at Send-Reply time.

### Shared components (all in `flowdrip_app.py`)
- `_is_4x4_graduate(contact, camp, responded_emails, enrolled_emails, dnc_emails)`
  (pure): True only when the campaign is `fourbyfour`, the contact's email
  is not in `responded_emails`, not in `dnc_emails`, and not in
  `enrolled_emails`. Unit-tested.
- `_get_or_create_jway_handoff_newsletter()`: returns the designated J Way
  handoff newsletter, creating it with `newsletter_style = "j_way"` from the
  rep's sector/region if none exists. Used by Path A and by Path B's
  "create new J Way" choice.
- `_enroll_contact_in_newsletter(newsletter_camp, contact)`: the single
  shared enroll worker, lifted from the existing Responses-page logic.
  Dedupes (never adds a contact already enrolled). Used by both paths.
- Path A: a completion hook in the send path that, after the final 4×4
  email sends, runs the gate and enroll for that contact.
- Path B: the per-contact "Add to newsletter" action wired into the
  responded list, calling the shared worker (and get-or-create for the
  new-J-Way choice).

## Build order (two slices)
1. **Slice 1 (low risk, reuses UI):** `_enroll_contact_in_newsletter`,
   `_get_or_create_jway_handoff_newsletter`, `_is_4x4_graduate`, and the
   Path B manual picker (existing-or-new). Ships standalone value.
2. **Slice 2 (touches send path):** the Path A completion hook that
   auto-enrolls non-responders when the last 4×4 email sends.

## Data flow
- Path A: scheduler sends last 4×4 email → load `responded.json`, DNC, and
  the handoff newsletter's enrolled set → `_is_4x4_graduate` → if True,
  `_enroll_contact_in_newsletter`.
- Path B: reply detected → `add_responded` (existing) → rep opens responded
  list → clicks "Add to newsletter" → picks existing or creates new J Way →
  `_enroll_contact_in_newsletter`.

## Error handling
- Any handoff/enroll failure is caught and logged; it never blocks the email
  send or crashes the scheduler.
- Dedupe and DNC checks apply on both paths.

## Testing
- `_is_4x4_graduate`: excludes responders, DNC, already-enrolled, and
  non-`fourbyfour` campaigns; includes a clean graduate.
- `_get_or_create_jway_handoff_newsletter`: creates with `j_way` style when
  none exists; reuses the existing one when present.
- `_enroll_contact_in_newsletter`: adds a new contact; is a no-op (dedupe)
  for a contact already enrolled.

## Out of scope
- A standing "designate the auto-handoff target newsletter" setting (Path A
  uses the zero-config default; add this later only if requested).
- The third-Tuesday blog/article touch.
- Any change to J Way newsletter content itself.
- Multi-newsletter auto-routing.
