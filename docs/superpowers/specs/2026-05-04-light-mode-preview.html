DripDrop Light Mode — Scope Preview

Open this file in a browser to see actual swatches and mockups. Everything in this preview is the proposed light-mode design (option A from chat — lifted white cards, accent via borders/strips). Dark mode is unchanged.

What's in scope

  • Default theme flip → light is the default for new visitors (existing users keep their saved preference).
  • 3 token contrast bumps in C_LIGHT: muted, border, and accent darkening.
  • Card elevation pattern applied to .fd-card / .fd-bub in light mode (subtle shadow + slightly darker border).
  • Newsletter / SlowDrip card palette restructured to white cards with colored left strip + darker accent in light mode. Dark mode untouched.
  • Today page activity panels (the right rail with "181 emails sending today" etc.) — same elevation treatment.
  • Dark mode polish (intentionally untouched)
  • Per-account theme persistence (stays browser-local for now)
  • Audit of every {fg}15 ad-hoc tint (follow-up if specific cards still look off after deploy)
  • Email editor / Quasar overrides (already covered by existing :root[data-theme="light"] rules)

Open decisions for you

  1. Are the 5 newsletter hues right? Look at the cards above (teal, blue, indigo, amber, cyan). Any that look wrong or any you want me to change?
  2. Enroll button color — currently each card's enroll button is its own identity hue (rich, varied). Alternative: standardize all enroll buttons to a single teal regardless of card hue (cleaner, less busy). Pick.
